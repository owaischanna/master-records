# Individual Record

Small Next.js dashboard that now persists monthly report data in Supabase.

Quick start:

1. Install dependencies

```bash
cd individual-record
npm install
```

2. Create a Supabase project and add a table named `individual_records` with these columns:

- `month` (text)
- `s_no` (text)
- `beneficiary_name` (text)
- `father_husband_name` (text)
- `cnic_number` (text)
- `village_name` (text)
- `union_council_name` (text)
- `investment_business_plan` (text)
- `amount` (numeric or text)
- `recovery_amount` (numeric or text)
- `receipt_number` (text)
- `date` (text)
- `deposit_slip_number` (text)
- `deposit_date` (text)
- `deposit_status` (text)
- `beneficiary_status` (text)
- `trade` (text)

3. Add your Supabase environment variables to Vercel or `.env.local`:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `SUPABASE_KEY` (optional)
- `SUPABASE_TABLE_NAME` (optional; defaults to `individual_records`)

4. Start the app:

```bash
npm run dev
```

## Supabase setup details

1. Create a Supabase project at https://app.supabase.com.
2. In the SQL editor, run the statements from `supabase-schema.sql` or use this schema:

```sql
create table if not exists public.individual_records (
  id bigint generated by default as identity primary key,
  month text not null,
  s_no text,
  beneficiary_name text,
  father_husband_name text,
  cnic_number text,
  village_name text,
  union_council_name text,
  investment_business_plan text,
  amount numeric,
  recovery_amount numeric,
  receipt_number text,
  date text,
  deposit_slip_number text,
  deposit_date text,
  deposit_status text,
  beneficiary_status text,
  trade text,
  inserted_at timestamp with time zone default now(),
  updated_at timestamp with time zone default now()
);

create unique index if not exists individual_records_month_sno_idx on public.individual_records (month, s_no);
```

3. Set your Supabase environment variables in Vercel or `.env.local`:

```bash
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
SUPABASE_ANON_KEY=your-public-anon-key
```

> Do not include `/rest/v1` in the `SUPABASE_URL`. It should be the project URL only, e.g. `https://xxxx.supabase.co`.

4. Import your existing Excel data into Supabase with the helper script:

```bash
npm run import-to-supabase
```

This imports every sheet in the Excel file, writes per-sheet CSV files to `data/csv`, and uploads the records to your Supabase table.

If you want to import only one sheet, pass the path and sheet name:

```bash
node scripts/import-excel-to-supabase.js data/data.xlsx "Master Database"
```

5. Deploy to Vercel and make sure the same env vars are set there.

## How the app now works

- The app reads monthly report rows from Supabase when environment variables are configured.
- Updates from the modal are sent to `pages/api/update-monthly-report.js` and saved into Supabase.
- Excel export is still available as a download from current data.
